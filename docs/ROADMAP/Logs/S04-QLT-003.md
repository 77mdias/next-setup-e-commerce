# S04-QLT-003 - Definir estrategia de cobertura por camada (unit + integration + e2e)

**Data:** 2026-03-10  
**Status:** ✅ Concluida

## Resumo

Foi definida e versionada a estrategia de cobertura por camada para os cenarios P0/P1 da Sprint 04.
O plano consolidou matriz `cenario x camada`, contrato minimo por layer (unit, integration, e2e), criterios de merge para branch protegida e referencia direta no pipeline de CI.

## Discovery realizado no codigo (impacto tecnico real)

- API critica (backend):
  - `src/app/api/checkout/route.ts` + `src/app/api/checkout/__tests__/route.integration.test.ts`
  - `src/app/api/webhooks/stripe/route.ts` + `src/app/api/webhooks/stripe/__tests__/route.integration.test.ts`
  - `src/app/api/orders/[orderId]/route.ts` + `src/app/api/orders/[orderId]/__tests__/route.integration.test.ts`
  - `src/app/api/orders/session/[sessionId]/route.ts` + `src/app/api/orders/session/[sessionId]/__tests__/route.integration.test.ts`
- Frontend/jornada:
  - `src/app/orders/success/page.tsx`, `src/app/orders/failure/page.tsx`
  - `src/app/orders/__tests__/page.integration.test.ts`
  - `src/hooks/useCheckout.helpers.test.ts` (baseline unit existente)
- Schema/persistencia:
  - Sem alteracao de schema Prisma nesta task.
  - A estrategia depende de transicoes de estado/pagamento ja modeladas (ex.: `OrderStatus`, `PaymentStatus`, `StripeWebhookEvent.status`).
- Integracoes externas:
  - Stripe checkout e webhook continuam com cobertura prioritaria em integracao.
  - E2E passa a ter contrato tecnico definido para reduzir flaky (retries, workers, artefatos).

## Entregas implementadas

- Matriz de cobertura por camada adicionada em `docs/ROADMAP/SPRINTS/sprint-04-qualidade-observabilidade.md`.
- Definition of Done de qualidade para merge adicionada na mesma sprint, com comandos por gate.
- Config de integracao atualizada para padrao de descoberta `src/**/__tests__/*.integration.test.ts` em `vitest.integration.config.ts`.
- Baseline de contrato E2E adicionada em `playwright.config.ts`.
- Scripts de execucao por camada adicionados em `package.json`:
  - `test:unit:critical`
  - `test:integration:critical`
  - `test:e2e`
- CI atualizado em `.github/workflows/ci.yml` com etapas:
  - `Unit critical baseline`
  - `Critical integration suite (P0)`
  - `E2E critical suite (conditional)`
