# S03-ADR-001 - Completar GET/POST/PUT/DELETE em /api/addresses com ownership estrito

**Data:** 2026-03-02  
**Status:** ✅ Concluída

## Resumo

API de enderecos evoluida para CRUD completo com validacao de payload, ownership estrito por `userId` e consistencia transacional da regra de endereco padrao unico.

## Implementacao aplicada

- `src/app/api/addresses/route.ts`:
  - implementados metodos `POST`, `PUT` e `DELETE` mantendo `GET`;
  - criado schema Zod para criacao/edicao com:
    - campos obrigatorios minimos,
    - limites de string,
    - validacao de CEP (`00000-000`),
    - validacao de payload estrito (`.strict()`).
  - ownership estrito aplicado nas mutacoes com filtro por `{ id, userId }`.
  - normalizacao de dados de endereco:
    - `zipCode` normalizado para formato `00000-000`,
    - `state` normalizado para uppercase,
    - `complement` vazio convertido para `null`.
  - regra de default unica por usuario em transacao (`db.$transaction`):
    - criacao do primeiro endereco promove default automaticamente;
    - mudanca de default desmarca anteriores;
    - remocao de endereco default promove substituto quando existir.
- `src/app/api/addresses/__tests__/route.integration.test.ts` (novo):
  - cobertura de auth (`401`), ownership (`404` para nao-owner), validacao (`400`) e consistencia de default em `POST/PUT/DELETE`.
