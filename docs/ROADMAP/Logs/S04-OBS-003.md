# S04-OBS-003 - Reduzir ruído operacional de warnings (hooks e imagens)

**Data:** 2026-03-11  
**Status:** ✅ Concluída

## Resumo

Foi executada auditoria operacional focada em warnings de hooks React e imagens nos fluxos críticos de compra. O baseline atualizado ficou sem warnings recorrentes nesses tópicos para `lint`, `build` e suíte crítica de testes.

Também foi formalizado o baseline de release desta task, com exceções explicitamente justificadas.

## Objetivo da task

Diminuir ruído de warnings de build/test para ampliar sinal operacional de incidentes reais, com foco em:

- dependências de hooks React;
- uso de imagens em checkout, pedidos e catálogo.

## Discovery técnico aplicado

### Escopo auditado

- `src/components/products/products-catalog.tsx`
- `src/components/orders/orders-page-content.tsx`
- `src/app/checkout/page.tsx`
- `src/app/[slug]/checkout/page.tsx`

### Mapeamento de impacto real

- Frontend: componente de catálogo, página de pedidos e checkout (rotas críticas de compra).
- API: sem alterações de contrato HTTP.
- Schema/Prisma: sem alterações.
- Integrações (Stripe/Auth/Webhooks): sem alterações de comportamento.

### Resultado da auditoria

- Hooks: sem warnings de `react-hooks/*` nas rotas críticas auditadas.
- Imagens: uso de `next/image` sem warnings recorrentes nos pontos críticos auditados.
- Build/lint limpos para o escopo da task.

## Baseline operacional de warnings (release)

### Aceitável

- `0` warnings recorrentes de hooks/imagens em:
  - `npm run lint`
  - `npm run build`
  - validação crítica de testes de compra

### Exceções justificadas

- Logs `warn/error` estruturados durante testes de integração de cenários negativos são **esperados** (ex.: transição inválida, falha simulada de infraestrutura) e não são classificados como ruído operacional desta task.

## Validações executadas

- ✅ `npm run build`
- ✅ `npm run test:unit`
- ✅ `npm run test:integration:critical`
