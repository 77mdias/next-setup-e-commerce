# S03-QA-002 - Cobrir navegacao canonica e fluxo admin remove-bg em integracao

**Data:** 2026-03-07  
**Status:** ✅ Concluida

## Resumo

A cobertura de integracao da Sprint 03 foi ampliada para consolidar navegacao canonica e endurecer o contrato do endpoint administrativo de remove-bg. O foco foi reduzir regressao em redirects legados e garantir respostas previsiveis para cenarios de autorizacao/erro operacional.

## Implementacao aplicada

- Criada suite `src/app/cart/__tests__/page.integration.test.ts` para validar redirect de `/cart` para `/carrinho` com e sem querystring.
- Expandida suite `src/app/orders/__tests__/page.integration.test.ts` com caso canonico de `orders/success` sem `session_id`, garantindo fallback estavel para `/orders`.
- Expandida suite `src/app/api/admin/remove-bg/__tests__/route.integration.test.ts` com caso de erro previsivel `404` para imagem de origem indisponivel.
- Ajustado contrato em `src/app/api/admin/remove-bg/route.ts` para mapear `404` com mensagem sanitizada e status explicito.
- Task `S03-QA-002` marcada como concluida em `docs/development/tasks/PHASE-03-experiencia-conta-pedido.md` com criterios de aceite checados.
- Documentacao de rotas atualizada em `docs/01-analise-aplicacao/03-rotas-ui-api.md` para refletir alias legado `/cart` e comportamento previsivel de erro no endpoint admin.
