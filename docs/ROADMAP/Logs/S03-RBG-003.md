# S03-RBG-003 - Corrigir persistencia de imagens processadas no fluxo admin

**Data:** 2026-03-07  
**Status:** ✅ Concluida

## Resumo

O fluxo `/${slug}/admin/remove-bg` foi alinhado ao contrato real da API. A persistencia de imagens processadas deixou de chamar `PUT /api/products/${slug}` (incompativel) e passou a usar endpoint administrativo dedicado por `productId`, eliminando `404/405` no salvamento.

## Implementacao aplicada

- Nova API administrativa para persistencia de imagens de produto:
  - `PUT /api/admin/products/[productId]/images` em `src/app/api/admin/products/[productId]/images/route.ts`.
  - Guardas de seguranca com `requireAdminAccess` (`401` sem sessao, `403` sem role admin).
  - Validacao de payload (`processedImages`) e normalizacao de entradas.
  - Retorno `404` para produto inexistente e `500` sanitizado para falhas internas.
- Atualizada tela admin `src/app/[slug]/admin/remove-bg/page.tsx`:
  - Leitura de produtos via contrato atual `GET /api/products`.
  - Persistencia via `PUT /api/admin/products/{productId}/images`.
  - Feedback consistente de sucesso/erro em UI e refresh da lista apos salvar.
- Ajustado `src/components/RemoveBgProcessor.tsx`:
  - Callback de persistencia apenas quando existe ao menos uma imagem processada com sucesso.
  - Mensagem explicita quando nenhuma imagem foi processada.
- Atualizada documentacao em `docs/04-setup-e-integracoes/REMOVE_BG_INTEGRATION.md` com os endpoints corretos e estrutura de arquivos vigente.
- Atualizada task em `docs/development/tasks/PHASE-03-experiencia-conta-pedido.md` (item, criterios e status).

## Evolucao adicional (2026-03-07)

- Escalabilidade do seletor admin em `src/app/[slug]/admin/remove-bg/page.tsx`:
  - paginação com `page/limit` (12 itens por página);
  - busca por nome de produto com campo dedicado;
  - indicadores de total de resultados, página atual e estado de atualização.
- `GET /api/products` passou a aceitar `query` para busca de nome (`contains`, case-insensitive):
  - filtro aplicado no `where` da consulta;
  - resposta passa a refletir `filters.query`;
  - chave de cache atualizada para incluir o termo de busca.
- Cobertura de integração atualizada em `src/app/api/products/__tests__/route.integration.test.ts` para garantir aplicação correta do filtro `query`.

## Riscos residuais

- A busca por nome usa `contains` case-insensitive. Em catálogos muito grandes, pode ser necessário otimizar índice/estratégia de busca no banco para manter latência baixa.
