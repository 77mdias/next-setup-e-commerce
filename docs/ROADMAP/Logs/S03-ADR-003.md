# S03-ADR-003 - Conectar endereco padrao/selecionado ao fluxo de checkout

**Data:** 2026-03-02  
**Status:** ✅ Concluida

## Resumo

Fluxo de checkout atualizado para consumir enderecos reais do usuario autenticado, selecionar endereco padrao automaticamente, permitir troca manual e recuperar estado em cenarios de endereco invalido/inacessivel sem quebrar navegacao.

## Implementacao aplicada

- `src/hooks/useCheckout.helpers.ts`:
  - adicionada deteccao de erro de endereco em respostas `400/404` (`isAddressCheckoutError`);
  - mensagens de erro de checkout agora retornam orientacao acionavel para recuperar selecao de endereco;
  - adicionada estrategia de fallback para selecao (`selectPreferredAddressId`): manter selecionado atual, fallback para default, depois primeiro endereco.
- `src/hooks/useCheckout.ts`:
  - carregamento de `GET /api/addresses` com estado dedicado (`addresses`, `selectedAddressId`, `isLoadingAddresses`, `addressError`);
  - selecao automatica do endereco padrao no carregamento;
  - envio de `addressId` efetivo no `POST /api/checkout` com validacao local de consistencia;
  - refresh automatico da lista de enderecos quando API retorna erro de endereco no checkout.
- `src/app/checkout/page.tsx`:
  - nova secao de endereco de entrega com selecao visual de endereco salvo;
  - fallback de UX para lista vazia (atalho para `/perfil`) e erro de carregamento com botao de retry;
  - submit bloqueado quando existe lista de enderecos, mas nenhum endereco valido esta selecionado.
- `src/app/api/checkout/__tests__/route.integration.test.ts`:
  - cobertura de persistencia de `addressId` valido no pedido;
  - cobertura de erro `400` para `addressId` invalido/inacessivel.
- `src/hooks/useCheckout.helpers.test.ts`:
  - cobertura de mapeamento de erro de endereco;
  - cobertura da logica de fallback de selecao de endereco.

## Riscos residuais

- O fluxo depende de disponibilidade de `/api/addresses`; em falha de rede/API o checkout exibe erro e exige retry para recuperar selecao de endereco.
- Nao foi criada suite de teste de interface para `src/app/checkout/page.tsx`; cobertura atual foca helper + integracao de API.
