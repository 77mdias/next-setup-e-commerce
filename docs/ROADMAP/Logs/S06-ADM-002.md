# S06-ADM-002 - Cobrir fluxo E2E critico do painel admin

**Status:** Concluida
**Data de conclusao:** 2026-03-18
**Estimativa original:** 7h
**Categoria:** Testes, homologacao e governanca

---

## Objetivo

Criar suite E2E com Playwright cobrindo os fluxos criticos do painel administrativo: controle de acesso, dashboard com KPIs, modulo de pedidos, modulo de catalogo/estoque e navegacao entre modulos.

## Decisoes tecnicas

1. **Arquivo unico de cobertura**: Criado `e2e/admin-critical-flow.spec.ts` com 8 cenarios organizados em 5 blocos tematicos, todos com tag `@critical` para execucao seletiva no CI.

2. **Reutilizacao de fixtures existentes**: Aproveitou o script `scripts/prepare-e2e-data.js` que ja cria admin user (ADMIN role), customer user (CUSTOMER role), store e produto com inventario.

3. **Cobertura de estados resilientes**: Os testes validam estados de loading, empty, error e sucesso sem depender de dados especificos de pedidos (que podem nao existir no ambiente E2E limpo).

4. **Infraestrutura ja existente**: Playwright config, CI workflow com artifact upload e scripts de preparacao ja estavam operacionais; nenhuma alteracao foi necessaria nesses arquivos.

## Cenarios implementados

### Bloco 1 - Controle de acesso (2 cenarios)

- Bloqueio de usuario CUSTOMER ao acessar `/admin` → redirect para `/status?reason=forbidden`
- Bloqueio de usuario nao autenticado → redirect para `/status?reason=auth-required`

### Bloco 2 - Dashboard (2 cenarios)

- Login admin e renderizacao de KPIs, contexto do shell (operador, papel, escopo) e links de navegacao
- Seletor de janela temporal (7d/30d/90d) sem quebra de interface

### Bloco 3 - Pedidos (2 cenarios)

- Navegacao para modulo de pedidos, verificacao de filtros (busca, status, periodo) e lista/empty state
- Interacao com filtro de status sem quebra de UI

### Bloco 4 - Catalogo (2 cenarios)

- Navegacao para catalogo, verificacao de listagem por loja e secao de CRUD
- Visibilidade da secao de ajuste de estoque para admin autorizado

### Bloco 5 - Navegacao cross-modulo (1 cenario)

- Navegacao sequencial Dashboard → Pedidos → Catalogo → Clientes → Auditoria → Dashboard sem links quebrados

- **Flakiness em CI com DB frio:** Os testes de dashboard e pedidos dependem de dados do `prepare-e2e-data.js`; cenarios de lista vazia sao tratados com fallback para empty state.
