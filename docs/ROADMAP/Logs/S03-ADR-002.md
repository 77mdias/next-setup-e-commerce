# S03-ADR-002 - Integrar CRUD de enderecos na tela de perfil (/perfil)

**Data:** 2026-03-02  
**Status:** ✅ Concluida

## Resumo

A tela de perfil passou a operar CRUD real de enderecos (criacao, edicao e remocao), eliminando placeholders de `toast` e mantendo sincronizacao da lista apos mutacoes.
Como mitigacao estrutural complementar, foi adicionada protecao no banco para impedir multiplos enderecos default por usuario.

## Implementacao aplicada

- `src/components/profile/profile-page-content.tsx`:
  - substituicao dos placeholders por chamadas reais para `POST/PUT/DELETE /api/addresses`;
  - carregamento e recarga da lista com estado consistente (`loading`, erro, sucesso);
  - suporte completo a criacao/edicao/remocao com feedback de UX;
  - listagem de todos os enderecos salvos com destaque de default e controle de remocao em progresso.
- `src/components/profile/address-form-modal.tsx` (novo):
  - modal de formulario para criacao/edicao com estado controlado;
  - validacao de cliente para campos obrigatorios e CEP;
  - toggle de endereco padrao e exibicao de erro de servidor.
- Mitigacao de risco estrutural de concorrencia/default:
  - `prisma/migrations/20260302203000_enforce_single_default_address_per_user/migration.sql` (novo):
    - normalizacao de defaults duplicados preexistentes;
    - criacao do indice parcial unico `addresses_userId_single_default_unique`.
  - `prisma/schema.prisma`: documentacao do indice parcial gerenciado por migration SQL.
