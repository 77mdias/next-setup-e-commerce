# S04-QLT-002 - Definir metricas minimas de saude e thresholds de alerta

**Data:** 2026-03-08  
**Status:** ✅ Concluida

## Resumo

Foi definida a baseline operacional de metricas minimas de saude para a Sprint 04, com SLI/SLO iniciais, thresholds de alerta e criterio objetivo de go/no-go para release.
O escopo cobre `checkout`, `webhooks/stripe`, `orders` e taxa de falha de pagamento, com fonte de coleta e responsavel operacional por metrica.

## Discovery realizado no codigo

- APIs criticas mapeadas para definicao de escopo:
  - `src/app/api/checkout/route.ts`
  - `src/app/api/webhooks/stripe/route.ts`
  - `src/app/api/orders/user/route.ts`
  - `src/app/api/orders/[orderId]/route.ts`
  - `src/app/api/orders/session/[sessionId]/route.ts`
- Fontes de dados para metrica de falha de pagamento e processamento webhook:
  - `prisma/schema.prisma` (`Order.paymentStatus`, `StripeWebhookEvent.status`)
- Sinais operacionais atuais:
  - respostas HTTP nas rotas criticas (`4xx/5xx`);
  - transicoes para `paymentStatus = FAILED` em webhook de pagamento;
  - deduplicacao/processamento de evento em `stripe_webhook_events`.
