# S05-AUT-002 - Alinhar UX de redefinicao de senha com politica unica

**Data:** 2026-03-11  
**Status:** ✅ Concluida

## Resumo

Foi corrigida a divergencia entre UI de reset e backend: o frontend de redefinicao agora usa a mesma politica oficial de senha aplicada nas rotas de autenticacao.

- validacao cliente de `newPassword` alinhada ao modulo compartilhado `password-policy`;
- feedback acionavel com checklist visual de requisitos;
- tratamento uniforme de erros da API com exibicao de `details` quando presentes.

## Discovery tecnico aplicado

- Impacto em frontend:
  - `ResetPasswordForm` aplicava regra antiga de minimo 6 caracteres e nao orientava os requisitos completos.
  - Erros retornados pelo backend podiam ser exibidos sem detalhamento acionavel no formulario.
- Impacto em API:
  - nenhum endpoint alterado nesta task.
- Impacto em schema/integracoes:
  - sem alteracoes.

## Decisoes tecnicas

- Reaproveitado modulo compartilhado `src/lib/password-policy.ts` na UI de reset.
- Exportada lista canonica `PASSWORD_POLICY_REQUIREMENTS` para evitar duplicacao de regra/mensagem no cliente.
- Checklist de requisitos renderizado no formulario para reduzir tentativa e erro.
