# Roadmap de Novas Features (Etapas e Sprints)

## Objetivo
Evoluir a aplicacao para um padrao de producao com foco em seguranca, confiabilidade de checkout, qualidade e experiencia do usuario.

## Estrutura
- `01-backlog-priorizado.md`: backlog funcional e tecnico ordenado por impacto.
- `sprint-01-fundacao-seguranca.md`: hardening e correcoes criticas.
- `sprint-02-checkout-confiavel.md`: consistencia total de checkout/pedido/pagamento.
- `sprint-03-experiencia-conta-pedido.md`: melhorias de conta, endereco e experiencia de pedidos.
- `sprint-04-qualidade-observabilidade.md`: testes, telemetria e maturidade operacional.

## Cadencia sugerida
- Sprint de 2 semanas.
- Demo no fim de cada sprint.
- Critico de seguranca sempre bloqueia release.
