# Documentacao da Aplicacao My Store

## Ordem Recomendada de Leitura
1. `01-analise-aplicacao/00-resumo-executivo.md`
2. `01-analise-aplicacao/04-seguranca.md`
3. `02-guia-desenvolvedor-novato/01-setup-ambiente.md`
4. `ROADMAP/SPRINTS/README.md`

## Estrutura
- `00-templates/`: modelos padrao para manter e expandir documentacao.
- `01-analise-aplicacao/`: auditoria tecnica completa (dados, arquitetura, seguranca, testes, rotas).
- `01-analise-aplicacao/referencias-modelagem/`: referencias historicas de diagrama e schema completo.
- `02-guia-desenvolvedor-novato/`: onboarding pratico para dev iniciando no projeto.
- `02-guia-desenvolvedor-novato/referencias/`: materiais extensos de apoio para criacao de projeto.
- `03-seguranca-governanca/`: requisitos obrigatorios, hardening e governanca operacional.
- `04-setup-e-integracoes/`: setup de ambiente e guias de integracao (OAuth, Stripe, Remove.bg, notificacoes).
- `ROADMAP/`: backlog, sprints, tarefas e logs tecnicos.

## Publicos
- Dev novato: comecar em `02-guia-desenvolvedor-novato/`.
- Dev backend/frontend: comecar em `01-analise-aplicacao/03-rotas-ui-api.md`.
- Tech lead/arquitetura: `01-analise-aplicacao/01-arquitetura.md` e `01-analise-aplicacao/06-riscos-priorizados.md`.
- Seguranca/DevOps: `03-seguranca-governanca/`.
- Produto/planejamento: `ROADMAP/`.
